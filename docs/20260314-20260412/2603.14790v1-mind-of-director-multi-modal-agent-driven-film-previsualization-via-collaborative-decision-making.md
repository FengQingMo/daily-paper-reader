## 摘要
我们提出了 Mind-of-Director，这是一个由多模态智能体驱动的电影预视觉化（film previz）框架，用于建模电影制作团队中的协作决策过程。给定一个创意想法，Mind-of-Director 能协调多个专门化智能体，在游戏引擎中生成预视觉化序列。该框架由四个协作模块组成：剧本开发（Script Development），智能体通过迭代撰写与完善剧本；虚拟场景设计（Virtual Scene Design），将文本转换为语义一致的三维环境；角色行为控制（Character Behaviour Control），负责确定角色调度与动作；以及镜头规划（Camera Planning），用于优化画面构图、运动与摄影效果。内置于游戏引擎的实时可视化编辑系统进一步支持交互式检查与跨场景、行为、镜头的同步时间线调整。大量实验与人工评估表明，Mind-of-Director 能在约25分钟内为每个创意生成高质量、语义扎实的预视觉化序列，验证了智能体协作在自动化原型设计与人机共创电影制作中的有效性。

## Abstract
We present Mind-of-Director, a multi-modal agent-driven framework for film previz that models the collaborative decision-making process of a film production team. Given a creative idea, Mind-of-Director orchestrates multiple specialized agents to produce previz sequences within the game engine. The framework consists of four cooperative modules: Script Development, where agents draft and refine the screenplay iteratively; Virtual Scene Design, which transforms text into semantically aligned 3D environments; Character Behaviour Control, which determines character blocking and motion; and Camera Planning, which optimizes framing, movement, and composition for cinematic camera effects. A real-time visual editing system built in the game engine further enables interactive inspection and synchronized timeline adjustment across scenes, behaviours, and cameras. Extensive experiments and human evaluations show that Mind-of-Director generates high-quality, semantically grounded previz sequences in approximately 25 minutes per idea, demonstrating the effectiveness of agent collaboration for both automated prototyping and human-in-the-loop filmmaking.

## 二、方法论概述

### 2.1 核心思想
- 引入**多模态多智能体协作系统**，由虚拟导演统筹多个专职智能体共同完成剧本撰写、虚拟场景建设、角色行为控制与摄像规划。
- 采用多轮协商决策机制，模仿现实影视创作过程中的“讨论—修订—评估”循环，以提升生成结果的一致性与艺术合理性。

### 2.2 四大模块及核心流程
1. **剧本开发（Script Development）**  
   - 将创意输入 \( I \) 转化为结构化剧本文本 \( S=g_1(I) \)。  
   - 三类智能体（编剧、演员、导演）形成 *Discuss–Revise–Judge* 循环：编剧初稿 → 演员讨论反馈 → 编剧修订 → 导演评判。
   - 输出包含角色设定、分幕描述与对话文本的完整剧本。

2. **虚拟场景设计（Virtual Scene Design）**  
   - 将剧本描述转化为三维场景 \( E=g_2(S) \)。  
   - **关键技术**：
     - 2D图像引导生成（利用 Florence2 + SAM 提取空间先验）。
     - 从 Objaverse 精选可编辑 3D 资产。
     - 场景图（Scene Graph）推理 + 分层占据网格布局实现合理物体安置。
     - LLM补充装饰元素以提高真实感。  
   - 结果为语义一致、可编辑的三维环境。

3. **角色行为控制（Character Behaviour Control）**  
   - 在场景内确定角色位置与动作 \( B=g_3(S,E) \)。  
   - **主要步骤**：
     - 通过可行区域优化（最小化碰撞与边界损失、最大化可见性）。
     - 多模态LLM结合视觉与语义提示推断角色的动作区域、朝向与姿势。
     - 从 Mixamo 动画库中选择并重定向匹配动作；利用 Unity NavMesh 实现物理可行路径。
     - 使用 Discuss–Revise–Judge 循环细化情节与动作一致性。

4. **摄像规划（Camera Planning）**  
   - 决定镜头模板与参数 \( V=g_4(S,E,B) \)。  
   - 建立包含 21 类镜头（单人、双人与群像）的模板库。  
   - 使用 *Debate–Judge–Validation* 循环：两位虚拟摄影师辩论 → 导演裁定 → 引擎验证物理合理性（碰撞、遮挡检测）。  
   - 输出最终镜头配置及摄影轨迹。

### 2.3 总体流程表达式
\[
V = G(I) = g_4(g_3(g_2(g_1(I))))
\]
该公式描述了从创意输入到预演视频输出的多阶段生成路径，其中 \( G \) 表示由所有模块组成的多智能体协作函数。

## 三、实验设计

### 3.1 数据集与基准
- **自建数据集 PrevizPro**：  
  - 10 个创意主题 × 每主题 3 幕 × 每幕约 12 个片段，共 **360 个片段**。  
  - 每个片段人工标注角色动作与镜头模板，用作定量评估参考。
  - 仅用于测试，框架本身为“无训练式”（training-free）。

### 3.2 对比基线
- **StageDesigner**［2025］：基于语言推理的剧本到3D舞台生成系统（仅场景设计）。
- **FilmAgent**［2024］：多智能体语言驱动的虚拟制作系统（仅行为与镜头规划）。
- 比较旨在验证多模态视觉先验与协作推理带来的改进。

### 3.3 实验指标
- **剧本模块**：角色真实性(CA)、节奏感(PA)、整体质量(OA)。  
- **场景模块**：CLIP文本-图像一致性、碰撞率、可交互性(Reach/Walk)、以及人工评估(VQ、PP、SA)。  
- **行为模块**：空间损失、位置合理性(Pos-P)、动作准确率与多样性。  
- **镜头模块**：碰撞率、遮挡率、镜头模板准确率与画面质量(OQ)。
