## 摘要
深度研究智能体（Deep Research Agents, DRAs）将大语言模型（LLM）的推理能力与外部工具相结合。记忆系统使得 DRAs 能够利用历史经验，这对于高效推理和自主进化至关重要。现有方法通常依赖从记忆中检索相似的推理轨迹来辅助推理，但存在记忆进化效率低以及存储和检索成本不断增加等关键问题。为了解决这些问题，我们提出了一种新的记忆智能体（Memory Intelligence Agent, MIA）框架，由管理者—规划者—执行者三部分组成。记忆管理者是一个非参数记忆系统，可以存储压缩后的历史搜索轨迹；规划者是一个参数化记忆智能体，用于为问题生成搜索计划；执行者则是另一个根据搜索计划进行信息搜索与分析的智能体。为了构建 MIA 框架，我们首先采用交替强化学习范式，以增强规划者与执行者之间的协作。此外，我们使规划者能够在测试阶段学习中持续进化，并在推理过程中实时更新而不打断推理流程。我们还建立了参数化记忆与非参数化记忆之间的双向转换循环，以实现高效的记忆进化。最后，我们结合反思机制和无监督判断机制，以提升在开放世界中的推理与自我进化能力。在十一项基准测试中的大量实验证明了 MIA 的优越性。

## Abstract
Deep research agents (DRAs) integrate LLM reasoning with external tools. Memory systems enable DRAs to leverage historical experiences, which are essential for efficient reasoning and autonomous evolution. Existing methods rely on retrieving similar trajectories from memory to aid reasoning, while suffering from key limitations of ineffective memory evolution and increasing storage and retrieval costs. To address these problems, we propose a novel Memory Intelligence Agent (MIA) framework, consisting of a Manager-Planner-Executor architecture. Memory Manager is a non-parametric memory system that can store compressed historical search trajectories. Planner is a parametric memory agent that can produce search plans for questions. Executor is another agent that can search and analyze information guided by the search plan. To build the MIA framework, we first adopt an alternating reinforcement learning paradigm to enhance cooperation between the Planner and the Executor. Furthermore, we enable the Planner to continuously evolve during test-time learning, with updates performed on-the-fly alongside inference without interrupting the reasoning process. Additionally, we establish a bidirectional conversion loop between parametric and non-parametric memories to achieve efficient memory evolution. Finally, we incorporate a reflection and an unsupervised judgment mechanisms to boost reasoning and self-evolution in the open world. Extensive experiments across eleven benchmarks demonstrate the superiority of MIA.

## 二、方法论与技术方案

### 1. 核心思想
MIA 引入一个受“大脑海马体结构”启发的三层结构——**Manager–Planner–Executor（管理者–规划者–执行者）**：
- **Memory Manager（管理者）**：非参数记忆模块，存储压缩后的历史搜索轨迹（workflow）；
- **Planner（规划者）**：参数化记忆模块，将经验迁移为内化能力（parametric memory）；
- **Executor（执行者）**：执行推理计划，负责检索、分析与回答问题。

这些模块共同组成一个**Plan–Execute–Reflect循环**，不断执行知识检索与经验抽象，实现自我优化和推理增强。

### 2. 记忆演化流程
MIA 的推理循环包括三个阶段：

1. **记忆检索 (Memory Retrieval)**  
   - 基于语义相似度、成功率（value reward）和使用频率（frequency reward）三维加权，检索历史轨迹；
   - 同时检索成功案例（正样本）与失败案例（负样本），为规划者提供对比性上下文。

2. **协作推理 (Collaborative Reasoning)**  
   - **Planner** 根据检索记忆生成行动计划（分步骤任务分解）；  
   - **Executor** 按计划执行，通过 ReAct 框架调用工具（如文本搜索与图像搜索），并向 Planner 反馈执行状态；  
   - 若评估结果不理想，触发 **Reflect-Replan** 机制重新规划一次（只触发一次以节省时间）。

3. **经验固化 (Experience Consolidation)**  
   - 通过 LLM Judger 评估结果；  
   - 由 Memory Manager 将冗长轨迹压缩成结构化摘要（image caption + workflow summary）；  
   - 将其存入记忆库或替换相似项目；同时更新 Planner 参数，实现显式（non-parametric）与隐式（parametric）联合进化。

### 3. 训练算法与优化框架

- **交替强化学习（Two-Stage Alternating RL）**  
  采用基于 GRPO（Group Relative Policy Optimization）的双阶段RL训练：
  1. **阶段 1**：固定 Planner，训练 Executor 理解和执行计划；
  2. **阶段 2**：固定 Executor，训练 Planner 学习利用记忆、反思与生成更优计划。  

- **奖励设计**：
  - Planner 奖励包含正确性、反思有效性、格式规范性等。
  - Executor 奖励包括回答正确性、工具调用成功率及格式约束。  
  - 奖励函数结合基于 LLM Judger 的定性与定量评估。

- **测试时学习（Test-Time Learning, TTL）**  
  推理时允许在线更新 Planner 参数，不中断推理流程。通过正负样本工作流对比强化策略，使模型实现在线自演化。

- **无监督自评机制（Unsupervised Judgment Framework）**  
  模仿学术审稿系统，引入三位“审稿人”分别评估逻辑一致性、信息可信度、结果有效性，并由“领域主席” (Area Chair) 综合判断，从而在无标注环境下提供稳定的训练信号。

### 2. 对比方法

- **直接回答 (Direct Answer)**：无工具；
- **搜索代理 (Search Agent/ReAct)**：通过 ReAct 框架；
- **记忆增强代理 (Memory-based Agent)**：对比 RAG、Mem0、A-Mem、ReasoningBank、ExpeL、Memento 等。  

MIA 方法对比了这些不同记忆机制下的性能。

### 3. 核心指标
主要采用**准确率 (Accuracy)**，由 Qwen3-32B 作为 LLM Judger 统一评估各模型的最终响应正误。

## 六、主要结论与发现

- **核心结论**：
  - MIA 在 **11 个基准任务中平均提升 5–9%**，刷新多项 SOTA；
  - 使用轻量级执行器（Qwen2.5‑VL‑7B）时，性能超过 Qwen2.5‑VL‑32B；
  - 对于 GPT‑5.4 等顶级闭源模型，MIA 仍可带来 6–9% 增益；
  - 无监督模式下效果接近监督版本，验证自演化机制有效；
  - 强化学习的交替优化显著增强了规划‑执行协同与策略泛化。

- **定性发现**：
  - 长上下文记忆方法在多轮任务中出现“噪声累积”；  
  - MIA 的双记忆机制（非参数+参数）可高效压缩与重用经验；
  - 在线 TTL 可在推理中持续演化模型—形成“self-evolving agent”。

## 七、优点与创新亮点

1. **架构创新**：Manager–Planner–Executor 层次化设计，实现认知‑操作‑存储的分离协同。  
2. **双记忆融合**：参数化与非参数化记忆相互转化，兼顾即时检索与长期内化。  
3. **交替强化学习策略**：分阶段优化 Planner 和 Executor 的互动行为。  
4. **在线自演化（TTL）**：推理时动态学习，首次实现在线迭代记忆更新。  
5. **无监督评审框架**：独创的“类同行评审”打分系统解决缺乏监督的问题。  
6. **小模型大性能**：Qwen‑7B 级别即可竞争闭源大模型表现。
