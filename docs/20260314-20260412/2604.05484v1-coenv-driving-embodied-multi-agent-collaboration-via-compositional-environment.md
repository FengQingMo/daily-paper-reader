## 摘要
多智能体具身系统在复杂的协作操控方面具有潜力，但在空间协调、时间推理和共享工作空间感知方面仍面临关键挑战。受人类协作中认知规划与物理执行分离的启发，我们提出了组合环境（compositional environment）的概念——一种真实世界与仿真组件的协同集成，使多个机器人智能体能够感知意图并在统一的决策空间中运作。基于这一概念，我们提出了CoEnv框架，它利用仿真实现安全的策略探索，同时确保在真实环境中的可靠部署。CoEnv包含三个阶段：将真实场景重建为仿真环境以数字化物理工作空间；基于视觉语言模型（VLM）的动作合成，支持通过高层接口进行实时规划以及通过代码驱动的轨迹生成进行迭代规划；以及带有碰撞检测的仿真到真实验证传递以保障安全部署。在具有挑战性的多机械臂协作操控基准测试中，大量实验表明CoEnv在实现高任务成功率与执行效率方面的有效性，确立了多智能体具身人工智能的新范式。

## Abstract
Multi-agent embodied systems hold promise for complex collaborative manipulation, yet face critical challenges in spatial coordination, temporal reasoning, and shared workspace awareness. Inspired by human collaboration where cognitive planning occurs separately from physical execution, we introduce the concept of compositional environment -- a synergistic integration of real-world and simulation components that enables multiple robotic agents to perceive intentions and operate within a unified decision-making space. Building on this concept, we present CoEnv, a framework that leverages simulation for safe strategy exploration while ensuring reliable real-world deployment. CoEnv operates through three stages: real-to-sim scene reconstruction that digitizes physical workspaces, VLM-driven action synthesis supporting both real-time planning with high-level interfaces and iterative planning with code-based trajectory generation, and validated sim-to-real transfer with collision detection for safe deployment. Extensive experiments on challenging multi-arm manipulation benchmarks demonstrate CoEnv's effectiveness in achieving high task success rates and execution efficiency, establishing a new paradigm for multi-agent embodied AI.

     - **迭代式模式（Iterative Mode）**：  
       基于 **Claude Code 代码智能体**，自动生成完整多臂控制程序，通过多轮仿真执行与结果反馈实现代码级迭代优化。

       - 成功轨迹形成知识库 \( D = \{(s_0^{(j)}, G^{(j)}, τ^{(j)})\} \) 供后续学习。

## 三、实验设计

### 1. 硬件与任务场景

- **硬件配置**：  
  - 双臂设置：2 台 Franka Research 3 手臂；  
  - 三臂设置：1 台 Franka Research 3 + AgileX Piper 双臂平台；  
  - 感知：2~3 台 RealSense D435i 摄像头提供多视角 RGBD 输入。  
  场景重建与仿真基于 **ManiSkill + SAPIEN** 物理引擎。

- **五个多智能体操控任务（多臂基准测试）**：
  1. Cube Stacking（方块堆叠）  
  2. Ball Pickup（足球抬升）  
  3. Transfer Cylinder（圆柱传递）  
  4. Place Cucumber（三臂摆放黄瓜入锅）  
  5. Brush Box（三臂刷盒协作）

### 2. 评价指标

- **子任务成功率 Si**：每个关键里程碑完成次数 /10；
- **总体成功率 SR**：最终任务成功率；
- 平均成功率（两个模式综合）用于衡量整体性能。

### 3. 结果与对比

- **总体表现**：  
  - 交互式模式成功率约 **50%**；迭代式模式约 **48%**；
  - 总体平均成功率 **49%**；
  - 最优任务 Cube Stacking 达 **75%** 成功，最难 Transfer Cylinder **25%**。

- **模式互补性**：
  - *交互式模式*：在需要实时适应的动态协作任务上优势明显（如 Transfer Cylinder、Brush Box）；
  - *迭代式模式*：在追求精准轨迹的任务上表现更佳（如 Cube Stacking、Ball Pickup）。

- **三臂协作结果**：
  - Brush Box（非同步协作）可达 60%；
  - Place Cucumber（同步协作高要求）仅 35%。

### 4. 消融实验（Ablation Study）

- **自适应相机控制**去除后，平均成功率从 50% 降至 30%；  
  - 受遮挡任务（Transfer Cylinder, Brush Box）失败率显著上升。  
- **检查点验证**去除后，成功率仅剩 20%；  
  - 错误状态传播，导致长时间任务崩溃。  
- **综合效果**：两机制相互补充，实现 2.5× 性能提升。

### 5. 数据生成与扩展实验

- 利用仿真验证阶段生成高质量多智能体轨迹数据：  
  - 迭代模式的单次会话平均生成 9~17.5 条可用轨迹；  
  - 环境重置开销仅占 token 预算的 10%~30%。

## 六、结论与主要发现

- **CoEnv 框架证明了“组合环境”理念的有效性**：通过仿真强化规划安全性，再映射到现实中执行，实现可扩展的多臂协作。

- 提出机制（自适应视角与检查点验证）是提升成功率的关键因素。
- 同时提供了**自动化多智能体数据生成管线**，为未来训练通用机器人策略提供新路径。
