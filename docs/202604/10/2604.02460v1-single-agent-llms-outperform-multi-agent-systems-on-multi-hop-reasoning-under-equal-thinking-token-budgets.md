## 摘要
近期研究报告显示，多智能体大型语言模型系统（MAS）在表现上具有较强性能，但这些提升往往受到测试时计算量增加的影响。当计算量被标准化时，单智能体系统（SAS）可以与MAS的性能相当，甚至超越MAS。然而，这种比较背后的理论基础和评估方法尚不清晰。本文基于数据处理不等式提出一个信息论论点，表明在固定的推理token预算下且上下文利用完美的情况下，单智能体系统的信息效率更高。这一观点进一步预测，当单智能体的有效上下文利用率受损或消耗更多计算资源时，多智能体系统可能变得更具竞争力。我们在三个模型族（Qwen3、DeepSeek-R1-Distill-Llama 和 Gemini 2.5）上进行受控实证研究，在匹配预算条件下比较SAS与多种MAS架构。结果表明，当推理token保持恒定时，SAS在多跳推理任务上持续达到或超越MAS的性能。除了整体性能，我们还对系统行为和评估方法进行了详细的诊断分析。我们发现，在API层面的预算控制（尤其是Gemini 2.5）以及标准基准测试中存在显著的伪差，这些因素可能夸大了MAS的表面性能提升。总体而言，我们的结果表明，对于多跳推理任务，许多被报道的多智能体系统优势更可能来源于未考虑的计算和上下文效应，而非体系结构本身的固有优点，并强调理解和明确控制智能体系统中计算量、上下文和协调之间权衡的重要性。

## Abstract
Recent work reports strong performance from multi-agent LLM systems (MAS), but these gains are often confounded by increased test-time computation. When computation is normalized, single-agent systems (SAS) can match or outperform MAS, yet the theoretical basis and evaluation methodology behind this comparison remain unclear. We present an information-theoretic argument, grounded in the Data Processing Inequality, suggesting that under a fixed reasoning-token budget and with perfect context utilization, single-agent systems are more information-efficient. This perspective further predicts that multi-agent systems become competitive when a single agent's effective context utilization is degraded, or when more compute is expended. We test these predictions in a controlled empirical study across three model families (Qwen3, DeepSeek-R1-Distill-Llama, and Gemini 2.5), comparing SAS with multiple MAS architectures under matched budgets. We find that SAS consistently match or outperform MAS on multi-hop reasoning tasks when reasoning tokens are held constant. Beyond aggregate performance, we conduct a detailed diagnostic analysis of system behavior and evaluation methodology. We identify significant artifacts in API-based budget control (particularly in Gemini 2.5) and in standard benchmarks, both of which can inflate apparent gains from MAS. Overall, our results suggest that, for multi-hop reasoning tasks, many reported advantages of multi-agent systems are better explained by unaccounted computation and context effects rather than inherent architectural benefits, and highlight the importance of understanding and explicitly controlling the trade-offs between compute, context, and coordination in agentic systems.

## 一、研究背景与核心问题

- **研究动机**：  
  当前多智能体大型语言模型系统（Multi-Agent LLM Systems, MAS）在复杂推理任务中展现出强性能，如规划型、多角色协作、辩论与集成模型等。然而，多项研究指出这些性能提升**与计算量增加强相关**，而非源自架构优势。  
- **核心问题**：  
  当推理计算预算（尤其是“思维 token”预算）被严格控制时，**单智能体系统（Single-Agent System, SAS）是否仍能与多智能体系统匹敌或超越？**  
- **研究目标**：  
  论文探索在**固定 reasoning token 预算下**，SAS 与 MAS 的理论性能差异与实际表现，并揭示造成性能偏差的根源（计算负载与上下文利用）。

## 二、方法论与理论框架

### 1. 核心思想
- 基于**信息论视角**提出理论论证：  
  通过 **数据处理不等式 (Data Processing Inequality)**，SAS 能在同等推理 token 条件下实现更高的信息效率，因为单智能体内的推理不需跨 agent 传递消息。
- **关键信息流模型**：
  - 定义：
    - \(Y\)：正确答案；
    - \(C\)：单智能体可用的完整上下文；
    - \(M = g(C)\)：多智能体间传递的消息。
  - 按数据处理不等式：
    \[
    I(Y;C) \ge I(Y;M)
    \]
    表明信息经过多智能体通信后会丢失部分互信息，因此：
    \[
    P_e(M) \ge P_e(C)
    \]
    即单智能体最小错误率低于多智能体。

### 2. 扩展：上下文退化模型
- 为解释现实模型中上下文利用不完美现象，引入**上下文退化变换** \(T_\alpha(C)\) ，表示信息噪声、删除、混淆等效应。
- 推导出：当单智能体有效上下文退化严重时（即 \(I(Y;\tilde C_\alpha)\) 降低），精心设计的 MAS 可以通过结构化分解、过滤或验证机制**在退化环境中变得竞争力更强**。

## 三、实验设计

### 1. 数据集与任务
- **FRAMES**（Krishna et al., 2025）：多跳世界知识问答任务。  
- **MuSiQue 4-hop**（Trivedi et al., 2022）：四步推理问答任务，测试多层逻辑关系。

### 2. 对比架构
共测试 **三类模型族**与 **五种 MAS 结构**。
- **模型族**：
  - Qwen3-30B-A3B  
  - DeepSeek-R1-Distill-Llama-70B  
  - Gemini-2.5（Flash / Pro 两版本）

- **比较方法**：
  - **SAS**：单次调用、内在思考链。
  - **SAS-L（长思维变体）**：加入结构化分析指令以促发更多内部推理。
  - **MAS 架构**：
    - Sequential（顺序规划型）
    - Subtask-parallel（并行分解型）
    - Parallel-roles（角色分工型）
    - Debate（辩论型）
    - Ensemble（候选合并型）
    
  模型间均采用**匹配的思维 token 预算 B**进行控制。

### 3. 评估标准
- 使用 **LLM-as-a-judge** 评价范式：由独立语言模型依据语义一致性判定答案正确性。
- 所有系统共享相同 rubric 与评估 prompt，确保比较公平。

## 六、主要结论与发现

1. **核心结论：SAS 在匹配推理预算下通常优于或至少不弱于 MAS。**
   - 几乎所有模型族和数据集上，SAS 精度最高或与最佳MAS架构无显著差异。
   - SAS 使用更少的思维 token 达到同等或更高准确率。

2. **上下文退化条件下的反转现象：**
   - 当上下文被严重遮盖或替换噪声污染时，MAS（尤其 Sequential）相对更稳健。
   - 显示出 MAS 在高噪声或长上下文失效时可恢复部分推理能力。

3. **计算伪差识别：**
   - Gemini API 的 token 统计存在明显**思维 token 膨胀**（报告值高于实际输出4～5倍）。
   - MAS 因多次调用生成更多可见思维文本，导致表面预算不一致。
   - 建议未来在研究中**明确区分“请求预算”与“实际消耗”**。

4. **性能趋势：**
   - 准确率随预算增加先上升后趋于平稳（>2000 tokens 后收益递减）。
   - Gemini-2.5-Pro 为整体最强模型；MuSiQue 数据集难度显著高于 FRAMES。
