## 摘要
我们提出了 QuadAgent，这是一种无需训练、基于视觉-语言输入引导的敏捷四旋翼飞行智能体系统。与以往的端到端或串行智能体方法不同，QuadAgent 通过异步多智能体架构将高层推理与低层控制解耦：前景工作流智能体（Foreground Workflow Agents）负责处理主动任务和用户指令，而后台智能体（Background Agents）执行前瞻性推理。该系统通过印象图（Impression Graph）维护场景记忆，该图是由稀疏关键帧构建的轻量级拓扑地图，并利用基于视觉的避障网络确保飞行安全。仿真结果表明，QuadAgent 在效率和响应性方面优于基线方法。实验证明，它能够理解复杂指令，推理环境，并以最高 5 米/秒的速度在拥挤的室内空间中自主导航。

## Abstract
We present QuadAgent, a training-free agent system for agile quadrotor flight guided by vision-language inputs. Unlike prior end-to-end or serial agent approaches, QuadAgent decouples high-level reasoning from low-level control using an asynchronous multi-agent architecture: Foreground Workflow Agents handle active tasks and user commands, while Background Agents perform look-ahead reasoning. The system maintains scene memory via the Impression Graph, a lightweight topological map built from sparse keyframes, and ensures safe flight with a vision-based obstacle avoidance network. Simulation results show that QuadAgent outperforms baseline methods in efficiency and responsiveness. Real-world experiments demonstrate that it can interpret complex instructions, reason about its surroundings, and navigate cluttered indoor spaces at speeds up to 5 m/s.

## 二、方法论与核心算法

### 2.1 总体思想

QuadAgent通过**异步多智能体架构**实现认知与控制的分层解耦。整个系统由两个层次组成：
- **认知层（Cognitive Layer）**：负责语言理解、任务规划和高层逻辑推理；
- **物理层（Physical Layer）**：负责运动控制、轨迹跟踪和避障行为，运行在无人机上。

### 2.2 架构组成

- **前景工作流智能体（Foreground Workflow Agents）**：
  - 包含**调度器（Orchestrator）**、**规划器（Planner）**、**执行器（Executor）**三部分。
  - 实现**挂起-恢复协议（Suspend-and-Resume Protocol）**：当执行器调用物理动作技能后立即让出控制权，使无人机能“飞行同时推理”（Chatting-while-Flying）。
  - 支持实时用户输入处理（包括信息查询与冲突指令重规划）。

- **背景智能体（Background Agents）**：
  - 负责**前瞻性推理（Look-ahead Reasoning）**，在任务队列中提前分析未执行的子任务；
  - 查询场景图信息并缓存上下文，以减少后续任务的停顿时间。

- **技能库（Skill Library）**：
  - **记忆技能（Mnemonic Skills）**：在“印象图”（Impression Graph）上进行语义匹配与检索；
  - **感知技能（Perceptual Skills）**：调用 VLM 对实时图像分析、读取无人机状态；
  - **导航技能（Navigation Skills）**：负责从当前节点到目标关键帧的路径规划，并生成非阻塞的轨迹事件。

### 2.3 印象图（Impression Graph）

- 表示形式：轻量级稀疏拓扑图 \( G_{imp} = (N, E) \)，节点为关键帧，边表示可穿越性。
- **关键技术流程**：
  1. **节点创建**：依据空间位姿与视觉特征变化（CLIP嵌入余弦相似度），仅在发生显著变化时注册新节点；
  2. **边建立**：通过**金字塔视锥体体积交叉判定（Frustum Intersection）**推断可通航性；
  3. **语义增强**：采用多深度掩模输入（Near/Far/Full）以减少VLM生成的幻觉，生成空间感知描述。
- 该图支持路径规划和目标定位，无需密集3D重建。

### 2.4 物理层与控制策略

- 基于可微分物理的**视觉避障控制网络**：
  \[
  a_t, h_t = π_θ(I_{depth}, s_t, h_{t-1})
  \]
  其中网络以 RGB-D 深度流输入，在 30 Hz 下生成连续控制动作 \(a_t\)。  
- 控制状态机包括：悬停（q_hover）、路径对齐（q_align_s/e）、避障（q_avoid）等阶段。
