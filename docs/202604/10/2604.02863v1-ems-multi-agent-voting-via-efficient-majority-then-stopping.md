## 摘要
多数投票是将多智能体的响应汇聚为最终决策的标准方法。然而，传统方法通常要求所有智能体在开始聚合之前完成推理，这会导致显著的计算开销，因为一旦达成多数共识，许多响应就变得冗余。在本研究中，我们将多智能体投票表述为一个考虑可靠性的智能体调度问题，并提出了一种高效的多数—停止（Efficient Majority-then-Stopping, EMS）方法，以提升推理效率。EMS 基于任务相关的可靠性对智能体进行优先排序，并在多数达成时立即终止推理流程。该方法包含以下三个关键组成部分：具体而言，我们引入了智能体置信度建模（Agent Confidence Modeling, ACM），利用历史表现和语义相似性来估计智能体的可靠性；自适应增量投票（Adaptive Incremental Voting, AIV），用于以早停方式顺序选择智能体；以及个体置信度更新（Individual Confidence Updating, ICU），用于动态更新每个贡献智能体的可靠性。在六个基准测试上的大量评估表明，EMS 能稳定地将平均调用的智能体数量减少 32%。

## Abstract
Majority voting is the standard for aggregating multi-agent responses into a final decision. However, traditional methods typically require all agents to complete their reasoning before aggregation begins, leading to significant computational overhead, as many responses become redundant once a majority consensus is achieved. In this work, we formulate the multi-agent voting as a reliability-aware agent scheduling problem, and propose an Efficient Majority-then-Stopping (EMS) to improve reasoning efficiency. EMS prioritizes agents based on task-aware reliability and terminates the reasoning pipeline the moment a majority is achieved from the following three critical components. Specifically, we introduce Agent Confidence Modeling (ACM) to estimate agent reliability using historical performance and semantic similarity, Adaptive Incremental Voting (AIV) to sequentially select agents with early stopping, and Individual Confidence Updating (ICU) to dynamically update the reliability of each contributing agent. Extensive evaluations across six benchmarks demonstrate that EMS consistently reduces the average number of invoked agents by 32%.

## 一、研究动机与背景

- **核心问题**：传统的多智能体投票（Multi-Agent Voting）要求所有智能体完成推理后再进行聚合，这导致显著的计算冗余。当已有多数意见形成时，剩余智能体的推理结果往往不再影响决策，只增加推理成本。  
- **研究背景**：多智能体系统（MAS）被用于提升大型语言模型（LLMs）的推理多样性和稳健性，如多智能体辩论、角色协作等。然而，这些方法通常关注性能提升而忽视了效率优化。  
- **研究目标**：在保持多数投票准确性的前提下，减少冗余的智能体调用，提高整个多智能体系统的推理效率。

## 二、方法论：EMS 框架与关键技术细节

### 1. 核心思想
将多智能体投票重新表述为一个**“可靠性感知的智能体调度问题”**，提出“高效多数–停止”(Efficient Majority-then-Stopping, EMS)机制：  
一旦达成多数共识，即刻停止剩余智能体的推理，从而减少推理次数与成本。

### 2. 算法总体结构
EMS 包括三个关键模块：

#### （1）智能体置信度建模（Agent Confidence Modeling, ACM）
- 目标：评分每个智能体在当前任务上的可靠性，用于排序投票顺序。  
- 两种可靠性指标：
  - **历史可靠性（Shi,j = cj / vj）**：统计该智能体在历史任务中与最终投票结果一致的比例。  
  - **语义可靠性（Sqi,j）**：计算当前任务与智能体过去成功任务的语义相似度（通过句嵌入向量余弦相似度实现）。  
- 将以上置信度排序得到优先投票序列 Ψi。

#### （2）自适应增量投票（Adaptive Incremental Voting, AIV）
- 机制：按照 ACM 排序结果依次调用智能体进行投票。  
- 多数阈值 τ = ⌈(N + 1)/2⌉。  
- 若某一答案获票数 ≥ τ，则立即停止投票并输出结果。  
- 若未达成共识，则继续启用下一个智能体。若所有智能体均投票完仍无共识，则退化为标准多数投票。

#### （3）个体置信度更新（Individual Confidence Updating, ICU）
- 对参与投票的每个智能体更新状态 Φj = (cj, vj, Hj)。  
- 若该智能体与最终结果一致，则：
  - cj ← cj + 1  
  - vj ← vj + 1  
  - Hj ← Hj ∪ {ϕ(qi)}（更新语义记忆）  
- 通过持续更新可靠性估计，使系统逐渐学习智能体的稳定性与专长。

## 六、主要结论与发现

- **效率提升显著**：EMS 平均减少约 **32% 的智能体调用**（例如从 9.0 降至约 6.1）。  
- **准确率保持稳定**：与完整多数投票 (Simple MV) 基本持平（86.38% vs 86.61%）。  
- **早停机制有效性验证**：随机早停仍可保持高性能，但若缺失多数验证（如 Fixed Random-5），准确率明显下降。  
- **排序策略影响显著**：可靠性排序明显提升早停效率；历史可靠性略优于语义可靠性。  
- **可扩展性良好**：增大智能体数量 N 提升准确率，但收益递减，早停机制可有效控制开销增长。

## 七、优点与亮点

- **创新表述**：将投票视为“智能体调度问题”而非静态聚合，突破传统架构。  
- **结构简洁且可泛化**：EMS 模块化设计（ACM+AIV+ICU）易于集成到现有多智能体系统中。  
- **显著效率优越**：在保持精度的同时节省约三分之一的计算。  
- **动态更新机制**：个体置信度持续优化，可长期适应不同任务。  
- **全面实验验证**：覆盖多个类型任务和多品牌模型，强度高，可靠性好。
