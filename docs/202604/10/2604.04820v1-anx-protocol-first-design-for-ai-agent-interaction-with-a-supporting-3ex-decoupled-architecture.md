## 摘要
人工智能代理（AI agents）作为自主的数字行动体，需要专属的原生协议。目前的实现方法包括GUI自动化和基于MCP的技能，但存在高token消耗、交互碎片化、安全性不足等缺陷，原因在于缺乏统一的顶层框架和关键组件，每个独立模块都存在不足。为解决这些问题，我们提出了ANX——一种开放、可扩展、可验证的代理原生协议及顶层框架，融合了CLI、Skill和MCP，通过协议创新、架构优化和工具补充来解决核心痛点。ANX的四项核心创新包括：1）代理原生设计（ANX Config、Markup、CLI），具有高信息密度、灵活性和强适应性，可减少token消耗并消除不一致；2）结合Skill灵活性的人人机交互，实现指令双重渲染，即既可作为代理可执行指令，也可作为人类可读界面；3）通过MCP支持按需的轻量级应用，无需预注册；4）通过ANX Markup实现机器可执行的标准操作流程（SOP），消除歧义，确保长周期任务及多代理协作的可靠性。作为该系列的首篇论文，我们重点介绍ANX的设计，呈现其包含ANXHub的3EX解耦架构，并提供初步的可行性分析及实验验证。ANX确保原生安全性：UI到核心的通信绕过LLM，使敏感数据不会进入代理上下文；仅由人类确认机制防止自动化滥用。针对Qwen3.5-plus和GPT-4o的表格填充实验表明，相比基于MCP的技能，ANX分别减少了47.3%（Qwen3.5-plus）和55.6%（GPT-4o）的token使用；相比GUI自动化，分别减少了57.1%（Qwen3.5-plus）和66.3%（GPT-4o）的token使用；执行时间分别缩短了58.1%和57.7%。

## Abstract
AI agents, autonomous digital actors, need agent-native protocols; existing methods include GUI automation and MCP-based skills, with defects of high token consumption, fragmented interaction, inadequate security, due to lacking a unified top-level framework and key components, each independent module flawed. To address these issues, we present ANX, an open, extensible, verifiable agent-native protocol and top-level framework integrating CLI, Skill, MCP, resolving pain points via protocol innovation, architectural optimization and tool supplementation. Its four core innovations: 1) Agent-native design (ANX Config, Markup, CLI) with high information density, flexibility and strong adaptability to reduce tokens and eliminate inconsistencies; 2) Human-agent interaction combining Skill's flexibility for dual rendering as agent-executable instructions and human-readable UI; 3) MCP-supported on-demand lightweight apps without pre-registration; 4) ANX Markup-enabled machine-executable SOPs eliminating ambiguity for reliable long-horizon tasks and multi-agent collaboration. As the first in a series, we focus on ANX's design, present its 3EX decoupled architecture with ANXHub and preliminary feasibility analysis and experimental validation. ANX ensures native security: LLM-bypassed UI-to-Core communication keeps sensitive data out of agent context; human-only confirmation prevents automated misuse. Form-filling experiments with Qwen3.5-plus/GPT-4o show ANX reduces tokens by 47.3% (Qwen3.5-plus) and 55.6% (GPT-4o) vs MCP-based skills, 57.1% (Qwen3.5-plus) and 66.3% (GPT-4o) vs GUI automation, and shortens execution time by 58.1% and 57.7% vs MCP-based skills.

## 一、研究背景与核心问题

### 1.1 问题起点  
AI 代理逐渐演化为具备自主目标和任务执行能力的数字行动体，被视为下一代互联网用户。然而，现有的交互模式（如 GUI 自动化和 MCP 协议）仍沿用以人为中心的范式，导致以下核心问题：  
- **高 token 消耗**：GUI 自动化需解析复杂 DOM、传输大量视觉和语义信息。  
- **交互碎片化与标准缺失**：无统一顶层架构，使 CLI、Skill、MCP 各自为政。  
- **安全性薄弱**：敏感数据需穿越 LLM 上下文，存在隐私泄露风险。  
- **语义歧义与执行不稳定**：自然语言描述的 SOP 不够精确，难以实现长周期任务调度与多代理协作。  

### 1.2 研究目标  
论文旨在提出一套**代理原生（agent-native）**的统一交互协议与体系架构，实现：
- **高效**（低 token 开销）、  
- **可验证**（结构化 SOP 表达）、  
- **安全**（隔离敏感数据与人机确认）、  
- **可扩展**（动态发现与多代理协作）的交互体系。

## 二、方法论与技术框架

### 2.1 总体思想  
提出 **ANX（AI Native eX）协议体系**，采用“**协议优先（protocol-first）**”与“**3EX（Expression–Exchange–Execution）解耦架构**”相结合的设计思想。  
主线思路：  
> 将任务表达、工具发现、任务执行三层分离，通过结构化语义（ANX Markup）与运行时智能调度（ANXHub）连接，实现统一、安全、可扩展的代理交互生态。

### 2.2 核心组成与组件  
**ANX 协议家族：**
- **ANX Config**：统一配置文件格式，支持人机共享、简化初始化与环境描述。  
- **ANX Markup**：结构化任务描述语言，消除自然语言歧义；支持敏感字段标记。  
- **ANX CLI**：跨平台命令行载体，实现无 UI 依赖的执行。  
- **ANX SOP**：基于 Markup 的标准操作流程规范，显式定义任务拓扑、条件、依赖与分支。  

**3EX 架构层次：**
1. **Expression 层**：负责任务表达与语义结构化；定义元数据与安全标注。  
2. **Exchange 层（ANXHub）**：充当动态应用市场，基于语义向量搜索发现可用工具或代理，支持“用后即弃（use-and-go）”机制。  
3. **Execution 层**：由 ANX Core、ANX CLI、ANX Node 构成，实现命令生成与执行结果回流。

**安全机制嵌入式设计：**
- **UI-to-Core 通信**：敏感数据直接从前端 UI 传至 Core，绕过 LLM。  
- **Human-only Confirmation**：关键操作需人类确认，无程序化退出通道。  
- **Protocol State Machine**：在交互生命周期中显式定义安全状态（`WAITING UI`, `CONFIRMING`）。

## 七、方法与实验的优点

- **统一结构标记语言（ANX Markup）**：彻底消除自然语言歧义，使 SOP 可机读。  
- **3EX 解耦设计**：实现任务表达、发现与执行的职责分离，降低复杂度。  
- **安全机制内生化**：UI-to-Core 数据隔离 + 人类确认 = 类零信任执行路径。  
- **跨模型稳健性**：不同 LLM 均保持类似性能提升幅度。  
- **零安装生态（ANXHub）**：建立“语义市场”替代传统技能库注册。
