## 摘要
我们提出了\textbf{Deep Researcher Agent}，这是一个开源框架，旨在使大型语言模型（LLM）代理能够全天候自主开展深度学习实验。与现有主要聚焦于论文写作或代码生成的AI研究助手不同，我们的系统涵盖了完整的实验生命周期：假设形成、代码实现、训练执行、结果分析以及迭代优化。该框架引入了三项关键创新：(1) \textbf{零成本监控（Zero-Cost Monitoring）} —— 一种在模型训练期间完全不产生LLM API费用的监控范式，仅依赖进程级检查和日志文件读取；(2) \textbf{双层固定大小记忆（Two-Tier Constant-Size Memory）} —— 一种记忆架构，其容量限制在约5K字符，无论运行时间多长，均可避免长期运行代理所面临的上下文无限膨胀问题；(3) \textbf{最简工具集的主从架构（Minimal-Toolset Leader-Worker Architecture）} —— 一种多代理设计，每个工作代理仅配备3–5个工具，从而将每次调用的token开销降低最多达73%。在持续超过30天的部署中，该框架在四个并行研究项目中自主完成了500多个实验循环，其中一个项目通过200多个自动化实验实现了较基线指标52%的性能提升——而平均LLM成本仅为每24小时周期0.08美元。代码可在 https://github.com/Xiangyue-Zhang/auto-deep-researcher-24x7 获取。

## Abstract
We present \textbf{Deep Researcher Agent}, an open-source framework that enables large language model (LLM) agents to autonomously conduct deep learning experiments around the clock. Unlike existing AI research assistants that focus on paper writing or code generation, our system addresses the full experiment lifecycle: hypothesis formation, code implementation, training execution, result analysis, and iterative refinement. The framework introduces three key innovations: (1) \textbf{Zero-Cost Monitoring} -- a monitoring paradigm that incurs zero LLM API costs during model training by relying solely on process-level checks and log file reads; (2) \textbf{Two-Tier Constant-Size Memory} -- a memory architecture capped at $\sim$5K characters regardless of runtime duration, preventing the unbounded context growth that plagues long-running agents; and (3) \textbf{Minimal-Toolset Leader-Worker Architecture} -- a multi-agent design where each worker agent is equipped with only 3--5 tools, reducing per-call token overhead by up to 73\%. In sustained deployments spanning 30+ days, the framework autonomously completed 500+ experiment cycles across four concurrent research projects, achieving a 52\% improvement over baseline metrics in one project through 200+ automated experiments -- all at an average LLM cost of \$0.08 per 24-hour cycle. Code is available at https://github.com/Xiangyue-Zhang/auto-deep-researcher-24x7.

## 二、方法论与核心技术

**Deep Researcher Agent** 框架包含三个创新模块，构成完整的 **THINK → EXECUTE → REFLECT** 循环：

### 1. Zero-Cost Monitoring（零成本监控）

- **核心思想**：在GPU训练阶段完全不调用LLM接口，只依赖系统级进程检测与日志读取来确认训练状态。  
- **技术细节**：  
  - 检查进程存活 (`kill -0 $PID`)。  
  - 监控 GPU 利用率 (`nvidia-smi`) 防止假死。  
  - 读取训练日志最后50行（供本地日志记录）。  
- **效果**：避免传统周期性 LLM 轮询造成的高额 Token 消耗，将监控阶段成本降至 0 美元。

### 2. Two-Tier Constant-Size Memory（双层固定大小记忆）

- **目的**：防止上下文无限增长导致性能衰退和成本膨胀。  
- **结构设计**：  
  - **Tier 1**：项目摘要（Project Brief），人类定义，固定 3000 字符。  
  - **Tier 2**：滚动日志（Memory Log），记录关键结果与最新决策，容量2000字符。  
- **约束公式**：  
  \[
  |M_t| \le |B|_{\max} + |L|_{\max} = 5000 \text{字符}, ∀t
  \]
- **内存压缩机制**：自动删除最旧记录，仅保留关键节点与近期上下文。

### 3. Minimal-Toolset Leader-Worker Architecture（最小工具集主从架构）

- **架构组成**：  
  - **Leader Agent**：负责全局决策与任务分派。  
  - **Worker Agents**（3类）：
    - Idea Agent：文献检索与假设形成。  
    - Code Agent：代码修改、训练执行与 dry-run。  
    - Writing Agent：结果总结与报告生成。  
- **最简工具集原则**：每个工作代理仅含3–5个工具，减少每次调用时的 Token 开销约73%。  
- **执行逻辑**：  
  每一轮循环包括 THINK、EXECUTE、REFLECT 子阶段，每个阶段触发必要的 LLM 调用，期间严格控制上下文长度和调用频率。

### 4. 安全与人类干预机制

- 强制 **Dry-Run** 校验：确保代码能运行后再正式训练（捕获形状错误、配置缺失等）。
