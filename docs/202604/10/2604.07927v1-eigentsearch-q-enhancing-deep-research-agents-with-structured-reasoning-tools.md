## 摘要
深度研究需要对网络证据进行推理以回答开放性问题，这是人工智能代理的核心能力。然而，许多深度研究代理仍依赖于隐式、非结构化的搜索行为，这会导致重复探索和脆弱的证据聚合。受Anthropic的“think”工具范式和信息检索文献中相关见解的启发，我们提出了Q+——一套查询与证据处理工具，通过引导查询规划、监控搜索进展以及从长网页快照中提取证据，使网络搜索更加有条理。我们将Q+集成到Eigent的浏览器子代理中，Eigent是一个开源、可投入生产的用于计算机操作的多代理系统，从而得到EigentSearch-Q+。在四个基准测试（SimpleQA-Verified、FRAMES、WebWalkerQA 和 X-Bench DeepSearch）中，Q+分别使GPT-4.1、GPT-5.1和Minimax M2.5模型后端的Eigent浏览器代理基准规模加权平均准确率提升了3.0、3.8和0.6个百分点。案例研究进一步表明，EigentSearch-Q+通过显式化搜索过程和证据处理，生成了更连贯的工具调用轨迹。

## Abstract
Deep research requires reasoning over web evidence to answer open-ended questions, and it is a core capability for AI agents. Yet many deep research agents still rely on implicit, unstructured search behavior that causes redundant exploration and brittle evidence aggregation. Motivated by Anthropic's "think" tool paradigm and insights from the information-retrieval literature, we introduce Q+, a set of query and evidence processing tools that make web search more deliberate by guiding query planning, monitoring search progress, and extracting evidence from long web snapshots. We integrate Q+ into the browser sub-agent of Eigent, an open-source, production-ready multi-agent workforce for computer use, yielding EigentSearch-Q+. Across four benchmarks (SimpleQA-Verified, FRAMES, WebWalkerQA, and X-Bench DeepSearch), Q+ improves Eigent's browser agent benchmark-size-weighted average accuracy by 3.0, 3.8, and 0.6 percentage points (pp) for GPT-4.1, GPT-5.1, and Minimax M2.5 model backends, respectively. Case studies further suggest that EigentSearch-Q+ produces more coherent tool-calling trajectories by making search progress and evidence handling explicit.

## 二、方法论：Q+ 框架与结构化推理机制

### 1. 核心思想

- **Q+** 是一种模块化推理工具集，目标是在不引入外部知识召回的前提下，让模型通过结构化的“思考记录”形式表达中间推理状态。
- 它主要增强了 **Eigent 框架**（一个多智能体计算机操作系统）中的 **浏览器代理（Browser Agent）** 功能，使搜索行为具备明确的计划性和可审计性。

### 2. 系统集成结构

- **Eigent** 包含四个子代理：
  1. 浏览器代理（Browser Agent）——执行搜索与网页访问；
  2. 开发者代理（Developer Agent）——执行代码与命令；
  3. 多模态代理（Multimodal Agent）——处理视频、音频与图像；
  4. 文档代理（Document Agent）——创建与编辑文档。
- **EigentSearch-Q+** 在浏览器代理中加入 **Q+ 模块**，以增强其：
  - 查询规划（query planning）；
  - 搜索状态管理；
  - 网页证据提取与搜索进展评估。

### 3. 关键技术细节

- **Query-processing（查询处理）工具集：**
  - `plan_next_searches`：根据已知知识空缺生成下一个搜索计划，支持查询重写与扩展；
  - `select_query_and_search`：从查询前沿（frontier）选定最佳候选并发起搜索。
  - 系统维护 **frontier（待搜集）** 与 **explored（已搜集）** 两个查询集，防止重复查询。
- **Evidence-processing（证据处理）工具集：**
  - `extract_relevant_details`：从网页快照中提取与问题相关的信息；
  - `analyze_search_progress`：判断当前证据是否足以回答问题，若不充分则触发进一步搜索。
- **推理机制特征**：
  - 所有Q+工具均为**“think-tool”式自工具（self-tool）**，仅记录推理痕迹，不访问外部数据；
  - 增强了系统的**可观察性（observability）**与**可解释性（auditability）**；
  - 不修改核心浏览器代理，实现**轻量、非侵入式增强**。
