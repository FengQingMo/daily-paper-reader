## 摘要
企业级多智能体 AI 系统每小时会产生数千次智能体间交互，但现有的可观测性工具只能捕获这些依赖关系，而无法执行任何控制。OpenTelemetry 和 Langfuse 能够采集遥测数据，但将治理视为事后分析问题，而非实时执行目标。其结果是形成了一个“观察但不行动”的鸿沟，使得策略违规只能在造成损害之后才被发现。我们提出了治理感知智能体遥测（Governance-Aware Agent Telemetry，GAAT）这一参考架构，用于在多智能体系统中闭环连接遥测采集与自动化策略执行。GAAT 包括：（1）扩展了 OpenTelemetry 的治理遥测架构（Governance Telemetry Schema，GTS），为其增加治理属性；（2）一个支持 OPA 兼容声明式规则、延迟低于 200 毫秒的实时策略违规检测引擎；（3）具备分级干预机制的治理执行总线（Governance Enforcement Bus，GEB）；以及（4）具备加密来源保证的可信遥测平面（Trusted Telemetry Plane）。

## Abstract
Enterprise multi-agent AI systems produce thousands of inter-agent interactions per hour, yet existing observability tools capture these dependencies without enforcing anything. OpenTelemetry and Langfuse collect telemetry but treat governance as a downstream analytics concern, not a real-time enforcement target. The result is an "observe-but-do-not-act" gap where policy violations are detected only after damage is done.   We present Governance-Aware Agent Telemetry (GAAT), a reference architecture that closes the loop between telemetry collection and automated policy enforcement for multi-agent systems. GAAT introduces (1) a Governance Telemetry Schema (GTS) extending OpenTelemetry with governance attributes; (2) a real-time policy violation detection engine using OPA-compatible declarative rules under sub-200 ms latency; (3) a Governance Enforcement Bus (GEB) with graduated interventions; and (4) a Trusted Telemetry Plane with cryptographic provenance.

## 二、方法论与核心架构

### 1. 总体思路
GAAT 将 AI 治理嵌入到多智能体运行时的可观测体系中，形成“监测—检测—决策—执行”的闭环反馈系统，在亚秒级（<200 ms）延迟下做出治理决策。

### 2. 核心技术模块
- **治理遥测模式（GTS，Governance Telemetry Schema）**  
  扩展 OpenTelemetry 数据结构，增加治理属性，包括数据分类（classification）、司法辖区（jurisdiction）、敏感度（sensitivity）、溯源（lineage）及验证状态（verified）。所有事件以 ECDSA P-256 加密签名。

- **实时策略检测引擎**  
  基于 **OPA（Open Policy Agent）v0.60** 声明式规则系统，支持并行规则组合 `π1‖π2` 的“最大动作-置信度”合成逻辑：  
  若任一规则输出 “deny”，则合成结果为全局拒绝，确保**单调强化原则（Enforcement Monotonicity）**。

- **治理执行总线（GEB）**  
  负责执行五级干预动作：  
  `L0 ALLOW → L1 ALERT → L2 FLAG → L3 REDIRECT → L4 QUARANTINE`  
  使用分级升级函数  
  \[
  escalation(v, H(a,t)) = \min(4, base(v) + ⌊|H|/k⌋)
  \]  
  其中 \(H(a,t)\) 为智能体违规历史窗口。若在窗口内违规次数超过上限，将触发**硬电路断路（Circuit Breaker）**，强制隔离智能体。

- **可信遥测平面（Trusted Telemetry Plane）**  
  构建全链路信任基础：  
  - PKI 认证体系、TPM/SGX 安全模块  
  - Bloom filter 防重放、HMM 模型检测遗漏（omission）事件  
  - Merkle 树不可篡改日志审计

## 六、主要结论与发现

- GAAT 在 12,000 条真实轨迹上达 **99.7% 违规阻止率（VPR）**，合成数据上为 98.3%（±0.7%）。  
- 检测延迟中位数仅 **8.4 ms**，端到端执行延迟 **127 ms**。  
- 相较最佳基线（NeMo Guardrails 78.8%），违规逃逸率下降 **97.6%**。  
- 证明了治理模型在形式上的三项收敛与确定性性质，并在蒙特卡洛验证中达到 99.91% 收敛率。  
- 结果表明：**跨智能体溯源**是发现复杂违规的关键，而传统单智能体防护架构无法捕获此类问题。
