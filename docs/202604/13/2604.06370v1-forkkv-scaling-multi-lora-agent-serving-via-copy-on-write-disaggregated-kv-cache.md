## 摘要
大型语言模型（LLMs）的服务范式正在迅速向复杂的多代理工作流转变，在这种范式下，专业化的代理在庞大的共享上下文中协同工作。尽管低秩适配（Low-Rank Adaptation, LoRA）使得这些专业化代理能够在单一基础模型上高效共存，但它在推理服务阶段引入了关键的内存占用瓶颈。具体而言，独立的 LoRA 激活导致代理之间的 Key-Value（KV）缓存产生分歧，使得传统的前缀缓存对共享上下文失效。这迫使系统维护冗余的 KV 缓存，迅速耗尽 GPU 容量并降低吞吐量。为解决这一问题，我们提出了 ForkKV——一个用于多 LoRA 代理工作流的服务系统，其核心基于操作系统中的一种新型内存管理范式：写时复制（Copy-on-Write, CoW）的进程派生（fork）。通过利用 LoRA 的结构特性，ForkKV 将 KV 缓存物理拆分为一个大规模共享组件（类似于父进程的内存页）和轻量的代理特定组件（子进程的内存页）。为支持该机制，我们提出 DualRadixTree 架构，使新派生的代理能够继承庞大的共享缓存，并对其轻量化的独有缓存应用 CoW 语义。此外，为确保高效执行，我们设计了 ResidualAttention，一种能够在片上 SRAM 中直接重建解耦 KV 缓存的专用内核。针对不同任务的多种语言模型与实际数据集的全面评估表明，ForkKV 在几乎不影响生成质量的情况下，能够实现当前最先进多 LoRA 服务系统最高 3.0 倍的吞吐量。

## Abstract
The serving paradigm of large language models (LLMs) is rapidly shifting towards complex multi-agent workflows where specialized agents collaborate over massive shared contexts. While Low-Rank Adaptation (LoRA) enables the efficient co-hosting of these specialized agents on a single base model, it introduces a critical memory footprint bottleneck during serving. Specifically, unique LoRA activations cause Key-Value (KV) cache divergence across agents, rendering traditional prefix caching ineffective for shared contexts. This forces redundant KV cache maintenance, rapidly saturating GPU capacity and degrading throughput.   To address this challenge, we introduce ForkKV, a serving system for multi-LoRA agent workflows centered around a novel memory management paradigm in OS: fork with copy-on-write (CoW). By exploiting the structural properties of LoRA, ForkKV physically decouples the KV cache into a massive shared component (analogous to the parent process's memory pages) and lightweight agent-specific components (the child process's pages). To support this mechanism, we propose a DualRadixTree architecture that allows newly forked agents to inherit the massive shared cache and apply CoW semantics for their lightweight unique cache. Furthermore, to guarantee efficient execution, we design ResidualAttention, a specialized kernel that reconstructs the disaggregated KV cache directly within on-chip SRAM. Comprehensive evaluations across diverse language models and practical datasets of different tasks demonstrate that ForkKV achieves up to 3.0x the throughput of state-of-the-art multi-LoRA serving systems with a negligible impact on generation quality.

## 一、研究背景与核心问题

### 背景
- 大型语言模型（LLMs）正在从“单模型聊天”转向**多代理协作工作流（multi-agent workflows）**，典型场景包括代码助手、工具调用、推理与规划系统等。
- 多代理系统往往共享巨大且静态的上下文（如完整代码库、系统提示），但在推理阶段根据任务需要会分化出不同的上下文分支。
- 为提升专用能力，每个代理通常通过 **Low-Rank Adaptation (LoRA)** 方式对同一基础模型进行低秩微调，以支持不同任务。

### 核心问题
- 传统 LLM 服务中通过 **前缀缓存（prefix caching）** 来避免重复计算；但在多 LoRA 代理环境中，不同 LoRA 适配器会导致 **KV 缓存（Key-Value Cache）不一致**。
- 即使文本输入相同，不同 LoRA 也会产生不同的激活，导致 KV 缓存 **无法共享**，从而：
  - 显著增加显存消耗；
  - 降低并发度与吞吐量；
  - 破坏上下文复用机制。

**目标问题**：如何在保持生成质量的同时，使多 LoRA 代理能够高效共享上下文并复用主要计算结果。

## 二、方法与技术框架：ForkKV 系统

### 1. 核心思想
借鉴操作系统的 **「写时复制（Copy-on-Write, CoW）」内存管理原理**，设计一种可分离且可共享的 KV 缓存布局，将传统的 KV 缓存拆分为：
- **bCache（base cache）**：模型主干权重投影部分，可视为「父进程的共享页」；
- **rCache（residual cache）**：LoRA 特定的低秩残差部分，可视为「子进程的私有页」。

从而：
- 多个代理可共享同一大规模 bCache；
- 仅需为个别代理维护小型 rCache；
- 避免巨量冗余内存占用。

数学上，LoRA 投影被重新分解为：
\[
Y = xW + xA_iB_i = bCache + rCache \times B_i
\]
由此导出了可分共享与分离管理的架构。

---

### 2. 核心组件与技术细节

#### （1）DualRadixTree：双层前缀树式缓存管理
- 将缓存物理划分为 **Base 树** 与 **Residual 树**：
  - Base 树按 token 序列（token_id）索引共享的 bCache；
  - Residual 树按 (token_id, agent_id) 索引独有的 rCache。
- 新代理派生时执行两步：
  1. **Prefix Match**：查找可继承的共享上下文；
  2. **Copy-on-Write**：为新代理分配轻量 rCache。
- 支持独立的 LRU 驱逐策略，实现灵活的内存回收和部分复用。

#### （2）ResidualAttention：融合式注意力内核
- 问题：分离的 bCache 与 rCache 需在注意力计算中动态重建，若在 HBM（大显存）中重建会导致高延迟和内存占用。
- 解决：在 **片上 SRAM** 内完成 KV 重建与注意力计算，流程包括：
  1. **流式加载与延迟 RoPE**：将 bCache 与 rCache 块流入 SRAM，在上升投影（Bₖ）后再施加 RoPE；
  2. **并行计算基与残差部分的注意力**；
  3. **矩阵结合律融合输出**：
     \[
     O = Softmax(QK^T)V = Softmax(QK^T)V_{\text{base}} + [Softmax(QK^T)V_{\text{res}}]B_v
     \]
     避免在循环中重复 LoRA 投影计算，提高 GPU 并行性与带宽利用率。
- 结合算法与硬件的编码策略，有效减少显存访问和计算冗余。
