## 摘要
深入研究需要对网络证据进行推理，以回答开放式问题，这是人工智能代理的核心能力。然而，许多深度研究代理仍依赖于隐式的、无结构的搜索行为，导致冗余探索和脆弱的证据聚合。受到 Anthropic 的“think”工具范式和信息检索文献中洞见的启发，我们提出了 Q+，一套查询与证据处理工具，通过指导查询规划、监控搜索进展并从长网页快照中提取证据，使网络搜索更加有条理。我们将 Q+ 集成到 Eigent 的浏览器子代理中——Eigent 是一个用于计算机使用的开源、可投入生产的多代理系统，从而形成 EigentSearch-Q+。在四个基准测试（SimpleQA-Verified、FRAMES、WebWalkerQA 和 X-Bench DeepSearch）上，Q+ 将 Eigent 的浏览器代理在基准规模加权平均准确率上分别提升了 3.0、3.8 和 0.6 个百分点（pp），对应 GPT-4.1、GPT-5.1 和 Minimax M2.5 模型后端。案例研究进一步表明，EigentSearch-Q+ 通过使搜索进展和证据处理显性化，能够生成更连贯的工具调用轨迹。

## Abstract
Deep research requires reasoning over web evidence to answer open-ended questions, and it is a core capability for AI agents. Yet many deep research agents still rely on implicit, unstructured search behavior that causes redundant exploration and brittle evidence aggregation. Motivated by Anthropic's "think" tool paradigm and insights from the information-retrieval literature, we introduce Q+, a set of query and evidence processing tools that make web search more deliberate by guiding query planning, monitoring search progress, and extracting evidence from long web snapshots. We integrate Q+ into the browser sub-agent of Eigent, an open-source, production-ready multi-agent workforce for computer use, yielding EigentSearch-Q+. Across four benchmarks (SimpleQA-Verified, FRAMES, WebWalkerQA, and X-Bench DeepSearch), Q+ improves Eigent's browser agent benchmark-size-weighted average accuracy by 3.0, 3.8, and 0.6 percentage points (pp) for GPT-4.1, GPT-5.1, and Minimax M2.5 model backends, respectively. Case studies further suggest that EigentSearch-Q+ produces more coherent tool-calling trajectories by making search progress and evidence handling explicit.

## 一、核心问题与研究背景

- **问题动机**：  
  当前“深度研究代理”（Deep Research Agents）要在开放网络环境中回答复杂问题，通常依赖大语言模型（LLM）进行推理和搜索。然而，主流系统的搜索行为多为**隐式、非结构化**：
  - 搜索查询缺乏规划；
  - 证据整合过程混乱；
  - 模型在大量网页内容中容易出现冗余探索、难以停止或判断信息充足性。

- **研究目标**：  
  改进代理在多轮搜索中的结构化推理能力，使其在信息检索、证据理解与决策阶段更透明、可控、连贯。论文提出了一套工具机制 **Q+（Query Plus）**，用于让代理能“显性化”地规划查询与审视证据，从而提升研究过程的鲁棒性和可观察性。

- **理论背景来源**：  
  - 借鉴 **Anthropic 的 “think” 工具范式**：以独立的工具形式记录和外化中间推理结果，而非隐含在模型的输出文本中。  
  - 集成 **Information Retrieval (IR)** 经典策略（如查询扩展、前沿管理、伪相关反馈）到智能体工具空间。

## 二、方法论与技术细节

- **总体思想**：  
  将显性推理模块化为工具调用过程，使代理在查询和证据处理阶段都能生成可审查的中间状态。Q+并不调用外部API，而是作为结构化思维支架（cognitive scaffolding）记录模型决策逻辑。

- **系统架构**：  
  Q+ 被集成到开源多代理系统 **Eigent** 的“浏览器子代理”（Browser Agent）中，形成 **EigentSearch-Q+**。

- **工具设计**：
  1. **查询处理工具（Query-Processing Tools）**
     - `plan_next_searches`：识别知识空缺，设计后续查询（含查询改写、扩展、分解）。
     - `select_query_and_search`：从候选集合中选择最佳查询并执行搜索。
     - 维护两套集合：`frontier`（待搜索查询）与 `explored`（已执行查询），避免重复检索。

  2. **证据处理工具（Evidence-Processing Tools）**
     - `extract_relevant_details`：从长网页快照中提取与问题相关的信息。
     - `analyze_search_progress`：评估当前证据是否足以回答问题，决定停或继续。

- **执行流程（算法逻辑）**
  1. 用户输入复杂问题。
  2. Q+ 工具规划初步搜索方向（生成查询前沿）。
  3. 执行选定查询，浏览网页并提取核心信息。
  4. 检查搜索进展，判断是否需要进一步分解或扩展查询。
  5. 汇总证据，生成最终回答。

- **非侵入性特点**：  
  Q+ 作为一套可选插件，不改变Eigent核心逻辑；开发者可模块化开启或关闭这些工具。
